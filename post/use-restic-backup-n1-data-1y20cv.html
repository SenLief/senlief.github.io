利用restic备份N1数据

最近搞定了ipv6，同时家里有台N1没跑满，就想把一些小的服务仍家里N1跑，但是不太放心N1的稳定性，所以决定利用restic定时备份到U盘，遂让Ai帮忙写了一个bash脚本。

需求

  • 按需挂载/卸载U盘

  • 能够停止/开启特定的docker容器

    • 不少容器不支持热备份数据，需要停止后再启动

  • 备份特定的文件夹

  • 仅保留最近的几个版本减少备份的体积

  • 较为完善的检查和通知功能

  • 成功和失败后能通过tg bot通知

依赖

  • docker
  • docker compose
  • curl
